# Getting Started

To download, install, and run **VirtualMetric Director**, follow the instructions below.

## System Requirements

**Director** supports the `amd64` and `arm64` versions of **Windows**, **Linux**, and **macOS**. Its minimum requirements are:

- **1** CPU core
- **150MB** of memory
- **250mb** of disk space

We recommend using a machine with at least **2 CPU cores**. As **Director** uses a vectorized pipeline, it can utilize all available cores, and having multiples cores will improve performance.

## Installation

**Director** is a single binary&mdash;installation does not require any dependencies or prerequisites&mdash;so you can easily download and place it anywhere on your system. It can be installed on a traditional server, on an on-premises virtual machine, or in the cloud. It can also be deployed as an Azure App for a serverless configuration.

You can install **Director** online or offline, depending on your system's internet access.

### Online

If your system has internet access, you can use our automated installation scripts for a quick setup.

<Tabs>
  <TabItem value="powershell" label="PowerShell" default>  
    To run as a background process:
    ```PowerShell
    iwr dl.vget.me | iex
    ```

    To install as a service:
    ```PowerShell
    $p="C:\Program Files\VirtualMetric"; iwr dl.vget.me | iex
    ```
  </TabItem>
  <TabItem value="bash" label="Bash">  
    To run as a background process:
    ```bash
    curl -sL dl.vget.me | bash
    ```

    To install as a service:
    ```bash
    curl -sL dl.vget.me | p="/opt/virtualmetric" bash
    ```
  </TabItem>
</Tabs>

The script will

- detect your operating system and architecture
- download the appropriate version
- install and configure the binary based on your selected method (service or background process)
- launch the executable

Once you have downloaded the binary, open a terminal as _Administrator_ on **Windows**, as _sudo_ on **Linux**, or as _root_ on **macOS**.

* Create a directory for the binary:
    <Tabs>
      <TabItem value="windows" label="Windows" default>
        `C:\Program Files\VirtualMetric` (recommended)
      </TabItem>
      <TabItem value="linux" label="Linux">
        `/opt/virtualmetric` (recommended)
      </TabItem>
      <TabItem value="mac" label="macOS">
        `/Applications/VirtualMetric` (recommended)
      </TabItem>
    </Tabs>
* Copy the downloaded binary to the newly-created directory
* Choose the running mode:
  * As a background process
    <Tabs>
      <TabItem value="windows" label="Windows">
        ```PowerShell
        .\vmetric-director.exe -background
        ```
      </TabItem>
      <TabItem value="linux" label="Linux">
        ```bash
        ./vmetric-director -background
        ```
      </TabItem>
      <TabItem value="macos" label="macOS">
        ```bash
        ./vmetric-director -background
        ```
      </TabItem>
    </Tabs>
  * As a service:
    <Tabs>
      <TabItem value="windows" label="Windows">
        ```PowerShell
        .\vmetric-director.exe -service install
        .\vmetric-director.exe -service start
        ```
      </TabItem>
      <TabItem value="linux" label="Linux">
        ```bash
        ./vmetric-director -service install
        ./vmetric-director -service start
        ```
      </TabItem>
      <TabItem value="macos" label="macOS">
        ```bash
        ./vmetric-director -service install
        ./vmetric-director -service start
        ```
      </TabItem>
    </Tabs>

## Post-Installation

After completing the above steps, verify that the service is running:

<Tabs>
  <TabItem value="windows" label="Windows">
    ```PowerShell
    Get-Service vmetric-director
    ```
  </TabItem>
  <TabItem value="linux" label="Linux">
    ```bash
    systemctl status vmetric-director
    ```
  </TabItem>
  <TabItem value="macos" label="macOS">
    ```bash
    launchctl list | grep virtualmetric
    ```
  </TabItem>
</Tabs>
