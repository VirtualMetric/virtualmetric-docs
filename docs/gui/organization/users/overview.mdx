# Users: Overview

The **Users** view allows administrators to control user accounts, permissions, and access within the **DataStream** platform. It provides the ability to create, edit, disable, and monitor user accounts across your organization.

---

:::note
The **Users** view can be accessed through the **Home**&nbsp;>&nbsp;**Organization** pane by clicking **Manage Users**, or through the hamburger manu (top left) by selecting **Organization**&nbsp;>**Users**.
:::

The view contains the following key elements:

- **Users table** - Displays Full Name, Email, User Role, and Status for each user
- **Search users** - Text field to quickly find specific users by name or email
- **Status filter** - Dropdown to filter users by status (All, Active, Passive)
- **Add new user** - Blue button to create new user accounts
- **Items per page** - Controls pagination settings (10, 25, 50 items)
- **Page navigation** - Shows current page and total pages

## Users Columns

The table lists all registered users in your organization with their essential information:

- **Full name** - First and last name of the user
- **Email** - Primary address for contact and sign-in for the user
- **User Role** - The role assigned to the user in data stream management
- **Status** - Status of the user (_Active_ or _Passive_)

## Actions Menu

On the right side of each row, there is a vertical ellipsis (&#x22EE;). This opens the **Actions** dropdown which lists the actions that can be performed on the user:

- **Manage user** - Navigates to the **Details** page for the user
- **Delete user** - Deletes the user after which the user loses access to the organization
- **Disable/Enable user** - Disabling the user blacks access; enabling the user restores access

The **Add new user** button initiates the process of creating a new user.

## User Details

This tab contains the information defined in the user's account:

- **Full name** - First and last name of the user
- **User email** - Primary point of contact for the organization with the user. Enter a current, valid address
- **Timezone preference** - User's locale setting for date and time. See <Topic id="gui-account-timezone">**this section**</Topic> for details
- **Language** - Language preference of the user. See <Topic id="gui-account-language">**this section**</Topic> for details
- **Authentication** - Set password for the user.
- **Select role and user permissions** - User role and assigned permissions. (See <Topic id="gui-role-permissions">**this section**</Topic> for details.) Default choices:
  - **User (Read Only)** - User has only read-only access≈ü to data streams
  - **Admin** - User has full access to data streams

## Access Details

This tab contains a log of the details of the activities carried out by the user while accessing data streams. The table displays the following information:

- **Email** - Primary point of contact for the user
- **User IP** - IP address assigned to the user during access
- **Object Type** - Type of object accessed
- **Object** - Name assigned to the object when it was created
- **Action** - Action performed on the mentioned object
- **Action Description** - Detailed description of the task performed during access
- **Date** - Date of access
