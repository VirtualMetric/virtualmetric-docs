# Audit

The **Audit** page displays a log of all user activities and system events. The interface shows chronological records in a table format with filtering and search capabilities.

## Interface Elements

### Top Controls

- **Filter**: Button to apply advanced filtering criteria
- **Search bar**: Text input to search for specific users or actions
- **Refresh button**: Icon to update the audit log display

### Audit Table

The main table displays audit records with these columns:

- **User**: Email address of the user who performed the action
- **User IP**: Source IP address of the action
- **Object type**: Type of system component (User, ContentHub, Pipeline)
- **Object**: Specific target of the action
- **Action**: Type of operation performed
- **Action Description**: Human-readable explanation
- **Date**: Timestamp when the action occurred

### Pagination Controls

- **Items per page**: Dropdown to select display count (10 items shown)
- **Page counter**: Shows current range (e.g., "1-10 of 21 items")
- **Navigation**: Page numbers ("1 of 3 pages")
