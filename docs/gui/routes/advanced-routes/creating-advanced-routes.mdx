# Creating Advanced Routes

Advanced routes use a two-step wizard to create routing configurations with detailed settings and optional route configuration code.

## Step 1: Route Details

The **Define Route Details** form contains the following fields:

### Required Fields
- **Route name**: Text input for unique route identifier
- **Route description**: Multi-line text area (0/100 character limit)
- **Filter (Optional)**: Text input for filtering conditions (shows information icon for help)
- **Default target**: Dropdown showing "/dev/null" as default option
- **Select pipeline**: Dropdown to choose from available pipelines (e.g., "Microsoft Sentinel Automation and Normalization Pack")

### Route Status
- **Route Status** toggle switch (Enabled/Disabled)
- **Cancel** and **Next Step** buttons at the bottom

## Step 2: Route Configuration

The **Configure Route (Optional)** step provides:

### Configuration Area
- **Text editor** with line numbers (starting with line 1)
- **Large text input area** for entering or pasting configuration code
- Message: "You can enter or paste your configuration here."

### Navigation Buttons
- **Back** button to return to Route Details
- **Add route** button to complete the creation process

## Example Configuration

When filling in the form fields, you might see:
- **Route name**: "Sentinel Target Route Test"
- **Route description**: "Sentinel Target Route Test" (26/100 characters)
- **Filter**: "device.type == 'firewall'" (example conditional filter)
- **Default target**: Selected target from dropdown
- **Select pipeline**: "Microsoft Sentinel Automation and Normalization Pack"
- **Route Status**: Enabled (toggle switched on)

2. **Click "Add new route"** - Opens the route creation wizard

The interface shows progress with **Route Details** (step 1) and **Route Configuration** (step 2) tabs at the top.
