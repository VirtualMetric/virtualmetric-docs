# Creating Devices

To create a Push device:

- Go to **Home** → **Fleet Management** → **Devices**
- Choose your device type from the sidebar (Syslog, HTTP, UDP, TCP, or eStreamer)
- Click **Create device**

## Setup Process

### Basic Information

- Enter a **Device name** (must be unique)
- Add optional **Description** for the device purpose
- Select one or more **Directors** to manage this device

### Configuration

Depending on your device type, configure:

**Network Settings:**
- **Port** number for the device to listen on
- **IP address** binding (usually 0.0.0.0 for all interfaces)
- **Protocol options** (UDP/TCP for Syslog devices)

**Security Settings (if available):**
- **TLS encryption** for secure connections
- **Authentication** requirements
- **Certificate** configuration

**Advanced Options:**
- **Connection limits** and timeouts
- **Preprocessing pipelines** for data transformation
- **Multiple addresses** for redundancy

### Review and Create

- Review all settings in the summary
- Click **Create** to finish

The new device appears in the devices list and starts listening for incoming data immediately.

## Common Configurations

### Syslog Device
- **Port**: 514 (standard) or custom
- **Protocol**: UDP (faster) or TCP (reliable)
- **Format**: Standard syslog message parsing

### HTTP Device
- **Port**: 8080 or custom
- **Methods**: POST, PUT supported
- **Authentication**: Optional API key or basic auth

### TCP/UDP Devices
- **Port**: Custom port assignment
- **Connection limits**: Based on expected load
- **Buffer sizes**: For optimal performance
