# Creating Targets

To create a new target,

- In the **Home** view, click **Fleet Management**&nbsp;>&nbsp;**Targets**

-or-

- From the hamburger menu on the top left, select **Fleet Management**&nbsp;>&nbsp;**Targets**

This will take you to the **Add Target** view.

## Add Target

The elements of the view:

- **Name** - Unique identifier for the target
- **Description** - Optional details about the target's purpose (up to 100 characters)
- **Target Status** - Enable/disable toggle (enabled by default)
- **Post-processing pipeline** - Optional pipeline to apply to data before sending
- **Target Status** - Enable/disable toggle (enabled by default)
- **Post-processing pipeline** - Optional pipeline to apply to data before sending

Click the blue **Next step** button at the bottom.

### General Settings

- **Name** - Unique identifier for the target (required)
- **Description** - Details of the target's purpose (up to 100 characters) (optional)
- **Target Status** - _Enable_/_Disable_ toggle (enabled by default) (optional)

For **Microsoft Sentinel** targets, configure the following **Azure** properties:

- **Subscription ID** - Your Azure subscription
- **Resource Group** - Resource group containing the Log Analytics workspace
- **Workspace ID** - Target Log Analytics workspace identifier
- **Authentication** - Service principal or managed identity credentials

### Properties

- **Format** - Dropdown to select the format for the data to be forwarded (Elastic, Splunk, ASIM, etc.)

---

At any point, you can exit target creation by clicking the **Cancel** button at the bottom.

After finishing making the required settings, click **Add target** to complete the process.
