# Creating Targets

To create a new target,

- In the **Home** view, click **Fleet Management**&nbsp;>&nbsp;**Targets**

-or-

- From the hamburger menu on the top left, select **Fleet Management**&nbsp;>&nbsp;**Targets**

This will take you to the **Targets** view.

Click the **Add new target** button, then select the desired target type by clicking its card. This opens the first tab of the target-creation flow for that type.

## General Settings

- **Name** (Required) - Unique identifier for the target
- **Description** (Optional) - A note describing the purpose of the target
- **Target Status** - Enable/disable target. By default, enabled
- **Post-processing pipeline** (Optional) - Select a pipeline to normalize the output from the dropdown. See <Topic id="normalization">Normalization</Topic> for details.

Click **Cancel** to exit without creating a target, or **Next step** to go to the next tab.

Other tabs contain settings that are target-specific. You can find their details below:

<Details summary="Console">

The Console target writes log messages to standard output and supports field normalization to various logging standards.

**Key Configuration Fields:**
- **Field Format** - Data normalization format (ECS, CIM, ASIM)

For complete configuration details, see <Topic id="targets-console">Console target</Topic> documentation.

</Details>

<Details summary="Microsoft Sentinel">

The Microsoft Sentinel target ingests log messages into workspace tables using Data Collection Rules (DCRs) with automatic table selection and field normalization.

**Key Configuration Fields:**
- **Azure Authentication** - Tenant ID, Client ID, Client Secret
- **Data Collection Rule** - Rule ID and endpoint configuration  
- **Stream Configuration** - Table mapping and data flow settings

For complete configuration details, see <Topic id="targets-microsoft-sentinel">Microsoft Sentinel target</Topic> documentation.

</Details>
