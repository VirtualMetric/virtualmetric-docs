# NATS

<span className="theme-doc-version-badge badge badge--secondary">Messaging</span><span className="theme-doc-version-badge badge badge--secondary">Experimental</span>

## Synopsis

Creates a NATS target for publishing log messages to a NATS messaging system. Supports field normalization to various logging standards.

## Schema

```yaml {1,3}
- name: <string>
  description: <string>
  type: nats
  pipelines: <pipeline[]>
  status: <boolean>
  properties:
    field_format: <string>
```

## Configuration

The following fields are used to define the target:

|Field|Required|Default|Description|
|---|---|---|---|
|`name`|Y||Target name|
|`description`|N|-|Optional description|
|`type`|Y||Must be `nats`|
|`pipelines`|N|-|Optional post-processor pipelines|
|`status`|N|`true`|Enable/disable the target|

### Processing

|Field|Required|Default|Description|
|---|---|---|---|
|`field_format`|N|-|Field normalization format: `ecs`, `cim`, or `asim`|

## Details

The NATS target enables publishing log messages to NATS messaging systems. NATS is a lightweight messaging system for cloud-native applications, microservices architectures, and IoT messaging.

The target supports field normalization to convert log data into standard formats like Elastic Common Schema (ECS), Common Information Model (CIM), or Advanced Security Information Model (ASIM) before publishing to NATS.

:::note
This target is currently marked as experimental and provides basic functionality for NATS integration.
:::

### Field Normalization

Field normalization helps standardize log data before sending it to NATS, ensuring consistent data formats that can be easily processed by downstream consumers.

## Examples

### Basic

Minimum configuration for sending logs to NATS:

```yaml
targets:
  - name: basic_nats
    type: nats
    status: true
```

### With Field Normalization

Configuration using ECS field normalization:

```yaml
targets:
  - name: ecs_nats
    type: nats
    properties:
      field_format: "ecs"
```

### With Pipeline

Using a custom pipeline for additional log processing:

```yaml
targets:
  - name: pipeline_nats
    type: nats
    pipelines:
      - enrich_logs
    properties:
      field_format: "asim"
```

:::warning
Currently, the NATS target has limited functionality and is intended for basic integration. Future versions will include more robust configuration options and features.
:::
