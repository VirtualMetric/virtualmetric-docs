# CLI

Both **Director** and **Generator** can be accessed through the **CLI**, and various parameters can be specified using their options. Also, configuration files in YAML format can provide fields for specifying other relevant properties.

For simple illustrations of how they work, we recommend reading the [**Tutorials**](/docs/tutorials/quick-start.mdx).

### Director

**Command**: `vmetric-director <flags>`

The command line interface offers the following options:

|Flag|Default|Description|
|---|---|---|
|`mode`|N/A|Selects a _mode_. This is an alias for pipelines (valid options are ... _pfx2pem_, _read_, _pipelines_, etc.?)|
|`version`|`false`|Shows the version|
|`path`||Sets the installation path (of what ...?)|
|`pfx2pem`||Converts the certificate PFX to [PEM](/docs/reference/pem.mdx) files|
|`validate`|`false`|Validates the YAML configuration|
|`example`|`false`|Shows configuration examples|
|`compare-version`||Compares versions (of what?)|
|`supervisor`|`false`|Starts the _Supervisor_ (link to the chapter)|
|`update`|`false`|Starts the update (what update?)|
|`sentinel`|`false`|Starts the **Microsoft Sentinel** CLI|
|`snmpwalk`|`false`|Starts the **SNMPWalk** (explain or link to the chapter)|
|`console`|`false`|Starts the console (which console?)|
|`service`||Controls services (what services?)|
|`servicename`||Defines the service name|
|`agentless`|`false`|Starts in _Agentless Mode_ (link to the chapter)|
|`background`|`false`|Starts in the background|
|`info`|`false`|Shows information (what information?)|
|`vpc`|`false`|Shows _Product Code_|
|`director`|`false`|Starts _Director_ (Confusing. Aren't we already using the director?)|

### Generator

**Command**: `vmetric-generator <flags>`

The command line interface offers the following options:

|Flag|Default|Description|
|---|---|---|
|`mode`|N/A|Selects a _mode_ (valid options are ...?)|
|`version`|`false`|Version of the program. If specified, exits after printing the version|
|`example`|`false`|Shows a configuration example. If specified, exits after printing the example|
|`address`|`127.0.0.1:514`|Network address, including the port number|
|`severity`|_Error_|Severity (of what?)|
|`message`|`specified message`|Test nessage|
|`count`|`1000`|Message count. Valid range is **1..?**|
|`interval`|`1`|Interval (of what?) in seconds. Valid range is **1..?**|
|`duration`|`300`|Total duration (of what?) in seconds|
|`now`|`false`|Calculate the duration starting from _now_|
|`path`||The path (of what?)|
