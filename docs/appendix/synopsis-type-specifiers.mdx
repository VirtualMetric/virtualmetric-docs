# Synopsis Type Specifiers

To configure its components&mdash;i.e. devices, processors, routes, and targets&mdash; **Director** uses YAML files. The type specifiers for the fields can be stated as _singular_ or _plural_. Plural values are represented as

* comma-separated lists within a pair of brackets&mdash;e.g.

```code
["A", "B", ...]
```

-or-

* multi-line lists with `-` indicating entries&mdash;e.g.

```code
- "A"
- "B"
- ...
```

---

<dl>
	<dt>**any**</dt>
	<dd>Any value that can be represented by alphanumeric strings</dd>

	<dt>**char**</dt>
	<dd> _sing_. A text character.</dd>
	<dd>_plu_. **chars...** Can be a quoted string in which case the string is interpreted as an array of characters.</dd>

	<dt>**date**</dt>
	<dd>A time value based on an internationally recognized format</dd>

	<dt>**enum**</dt>
	<dd>
	A value selected from a specific number of pre-defined options&mdash;e.g. locale identifiers such as `en` or `fr`, date format specifiers such as `d`, `M`, or `y`  etc.

	Can be _ident_ or _string_.
	</dd>

	<dt>**id**</dt>
	<dd>Identity value for a type. `""` for strings, `0` for numeric values, `[]` for lists.</dd>

	<dt>**ident**</dt>
	<dd>_sing_. Identifier or unquoted alphanumeric string. Must start with an alphabetic character (`A..Z` or `a..z`), can contain alphabetic or numeric characters (`0..9`), `_` or `-` , and cannot contain spaces</dd>
	<dd>_plu_. **idents...** One or more identifiers.</dd>

	<dt>**ip-**</dt>
	<dd>
	A sequence of 4 numbers, each in the `0..255` range, separated by dots (`.`) representing the network identification of any device communicating with _Internet Protocol_&mdash;e.g.
		```code
			127.0.0.1
		```

	Can be _numeric_ or _string_. If it is a _string_, it must be quoted
	</dd>

	<dt>**numeric**</dt>
	<dd>Any value that can be represented by numeric characters (`0...9`)</dd>

	<dt>**logical**</dt>
	<dd>A boolean value, i.e. `true` or `false`</dd>

	<dt>**processor**</dt>
	<dd>_sing_. One of the available processors.</dd>
	<dd>_plu_. **processors...** One or more processors.</dd>

	<dt>**regex**</dt>
	<dd>
	A regular expression&mdash;e.g.
		```code
		_ingest.vendor.id =~ "fr-\w+\s+"
		```
	</dd>

	<dt>**string**</dt>
	<dd>_sing_. Quoted string. Can contain any text character.</dd>
	<dd>_plu_. **strings...** One or more quoted strings separated by whitespace (e.g. &blank;) or a delimiter (e.g. `,`).</dd>

	<dt>**script**</dt>
	<dd>
	A quoted string containing expressions that can be evaluated by an interpreter&mdash;e.g.
		```code
		_ingest.vendor.ip != null
		```
	It can be either an inline script or the name of a function invoked from a visible library.
	</dd>

	<dt>**shape**</dt>
	<dd>A standard shape description, such as `GeoJSON`, used for maps.</dd>

	<dt>**text**</dt>
	<dd>Unquoted string. Can contain alphanumeric characters and spaces</dd>
</dl>
