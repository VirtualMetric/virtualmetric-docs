# Example: Using Agents

This section shows how a typical **Agent** is configured.

## Director Configuration

Add a device configuration for **Director**:

```yaml
devices:
  - id: 1
    name: sample_agent
    type: windows
    status: true
    properties:
      type: agent 
    definitions:
      - id: 491591344758837249
      - id: 486227243115741541
```

Definitions are predefined Windows log channel configurations:

- `491591344758837249` is the base definition id
- `486227243115741541` is for security

There are 3 different security definitons that can be used under the `<vm_root>\package\definitions\module\windows\host\event` directory.

1. `windows_security_log_collector_all`: This is to collect all "Security" related events, "Microsoft-Windows-AppLocker/EXE and DLL", and "Microsoft-Windows-AppLocker/MSI and Script".

2. `windows_security_log_collector_common_example`: This is to collect specific events from "Security", "Microsoft-Windows-AppLocker/EXE and DLL", and "Microsoft-Windows-AppLocker/MSI and Script" channels.

## Agent Configuration

Download `vmetric-agent.exe` to a folder. Then, create a `vmetric.yml` file in the same folder, and add the following configuration to it:

```yaml
device:
  id: 1
director:
  address: 192.168.1.200:8080
```

Here

- the `device.id` value will be the same as that is configured in **Director**'s `devices` section.
- the `director.address` value is the IP address for **Director**'s service server; replace with your server's
- the listening port is `8080`
