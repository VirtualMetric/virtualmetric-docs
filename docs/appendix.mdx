# Appendix

## Synopsis Key

* **any** Any value of any type

* **char** Any text character

* **chars...** One or more characters. Can be a quoted string in which case the string is interpreted as an array of characters

* **date** A time value based on an internationally recognized format

* **enumerated** One of a limited number of options&mdash;e.g. locales such as `en`, `fr`, etc.

* **ident** Identifier or unquoted alphanumeric string. Must start with alphabetic uppercase or lowercase characters `A`-`Z` or `a`-`z`, can contain alphabetic or numeric characters `0`-`9`, and `_` or `-`

* **idents...** One or more identifiers.

* **logical** A boolean value, i.e. `true` or `false`

* **processors...** One or more of the available processors.

* **regex** A regular expression&mdash;e.g.
	```code
	_ingest.vendor.id =~ "locale-\w+\s+"
	```

* **string** Quoted string. Can contain any text character

* **strings...** One or more quoted strings. Can be
	* placed within a pair of brackets (`[]`) and separated by a space (` `) or a comma (`,`)&mdash;e.g.
		```code
			["A", "B", ...]
		```
	* listed on a line basis with a `-` at the beginning&mdash;e.g.
		```code
			- "A"
			- "B"
			- ...
		```

* **script** A quoted string containing syntactically valid expressions that can be run by an interpreter&mdash;e.g.
	```code
	if (_ingest.vendor.ip != null) {
		_ingest.network_direction = "inbound"
	}
	```
