# Remove

## Synopsis

```yaml {2}
remove:
   - field: <ident>
   - description: <string>
   - if: <script>
   - ignore_failure: <logical>
   - ignore_missing: <logical>
   - on_failure: <processors...>
   - tag: <string>
```

## Description

**Functionality**: Removes existing fields. If a field doesnâ€™t exist, an exception is raised.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y|N/A|List of fields to remove|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`ignore_missing`|N|`false`|If `true` and `field` does not exist or is `null`, exit quietly without making any modifications|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`tag`|N|-|Identifier|

## Examples

We have three fields: _foo_, _bar_, and _baz_.

<Example>
   <Description>
      We want to remove _foo_:
   </Description>
   <SampleCode>
      ```yaml
      - remove:
         field: foo
      ```
   </SampleCode>
   <Description>
      We want to remove _foo_ and _baz_:
   </Description>
   <SampleCode>
      ```yaml
      - remove:
         field: [foo, baz]
      ```
   </SampleCode>
</Example>
