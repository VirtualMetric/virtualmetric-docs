# Set

## Synopsis

```yaml {2}
set:
   - field: <ident>
   - copy_from: <enumerated>
   - description: <string>
   - if: <script>
   - ignore_empty_value: <logical>
   - ignore_failure: <logical>
   - media_type: <enumerated>
   - on_failure: <processors...>
   - on_success: <processors...>
   - override: <logical>
   - tag: <string>
   - value: <any>
```

## Description

**Functionality**: Sets a field and associates it with a specified value.

If the field already exists, its value will be replaced with the provided one.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y|N/A|Field to update|
|`copy_from`|N|-|Field whose value will copied to `field`. Cannot be used if `value` is specified. Valid options: `array`, `date`, `boolean`, `number`, `object`, `string`, etc.|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_empty_value`|N|`false`|If `true` and `value`/`copy_from` is not `null`, quietly exit if specified field does not exist or its value evaluates to `null` or `empty`|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`media_type`|N|`application/json`|For encoding `value`. Valid options: `application/json`, `text/plain`, or `application/x-www-form-urlencoded`.|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`on_success`|N|-|See [Handling Success](../handling-success.mdx)|
|`override`|N|`true`|Update non-null fields with `field`. If `false`, do not touch those fields|
|`tag`|N|-|Identifier|
|`value`|N|-|The value to be set. Cannot be used if `copy_from` is specified|

## Examples

* Assigning scalar values is straightforward:

<ExampleTable>
   <Description>
      The data contains a **foo** field:
   </Description>
   <SampleCode>
      ```json
      {
         "foo": 5
      }
      ```
   </SampleCode>

   <Description>
      Set its value to **1**:
   </Description>
   <SampleCode>
      ```yaml
      set:
         field: foo
         value: 1
      ```
      ```json
      {
         "foo": 1
      }
      ```
   </SampleCode>
</ExampleTable>

<ExampleTable>
   <Description>
      The **data.foo** field contains _A_:
   </Description>
   <SampleCode>
      ```json
      {
         "data": {
            "foo": "A"
         }
      }
      ```
   </SampleCode>

   <Description>
      Set its value to that of the **B** meta field:
   </Description>
   <SampleCode>
      ```yaml
      - set:
         field: data.foo
         value: {{{B}}}
      ```
      ```json
      {
         "data": {
            "foo": "META-B"
         }
      }
      ```
   </SampleCode>
</ExampleTable>

* Array fields can be accessed with the dot notation:

<ExampleTable>
   <Description>
      The **foo** field contains 3 elements:
   </Description>
   <SampleCode>
      ```json
      {
         "foo": ["A", "B", "C"]
      }
      ```
   </SampleCode>
   <Description>
      The _1st_ is copied to _bar_:
   </Description>
   <SampleCode>
      ```yaml
      - set:
         field: bar
         value: {{{foo.1}}}
      ```
      ```json
      {
         "foo": ["A", "B", "C"],
         "bar": ["A"]
      }
      ```
   </SampleCode>
</ExampleTable>

* The contents of a field containing complex values&mdash;such as arrays or objects&mdash;can be copied to another using `copy_from`:

<ExampleTable>
   <Description>
      The values in **foo**...
   </Description>
   <SampleCode>
      ```json
      {
         "foo": ["A", "B"]
      }
      ```
   </SampleCode>
   <Description>
      ...are copied into **bar**:
   </Description>
   <SampleCode>
      ```yaml
      - set:
         field: bar
         copy_from: foo
      ```
      ```json
      {
         "foo": ["A", "B"],
         "bar": ["A", "B"]
      }
      ```
   </SampleCode>
</ExampleTable>
