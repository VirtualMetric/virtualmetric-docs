# Set

## Synopsis

```yaml {2}
set:
  -  field: <ident>
  -  copy_from: <enumerated>
  -  description: <string>
  -  if: <script>
  -  ignore_empty_value: <logical>
  -  ignore_failure: <logical>
  -  media_type: <enumerated>
  -  on_failure: <processors...>
  -  on_success: <processors...>
  -  override: <logical>
  -  tag: <string>
  -  value: <any>
```

## Description

**Functionality**: Sets a field and associates it with a specified value.

If the field already exists, its value will be replaced with the provided one.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y|N/A|Field to update|
|`copy_from`|N|-|Field whose value will copied to `field`. Cannot be used if `value` is specified. Valid options: `array`, `date`, `boolean`, `number`, `object`, `string`, etc.|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_empty_value`|N|`false`|If `true` and `value`/`copy_from` is not `null`, quietly exit if specified field does not exist or its value evaluates to `null` or `empty`|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`media_type`|N|`application/json`|For encoding `value`. Valid options: `application/json`, `text/plain`, or `application/x-www-form-urlencoded`.|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`on_success`|N|-|See [Handling Success](../handling-success.mdx)|
|`override`|N|`true`|Update non-null fields with `field`. If `false`, do not touch those fields|
|`tag`|N|-|Identifier|
|`value`|N|-|The value to be set. Cannot be used if `copy_from` is specified|

## Examples

* Copying scalar values is straightforward:

<Example>
   <Description>
      Set value of the `foo` field to **1**:
   </Description>
   <SampleCode>
   ```yaml
   set:
      field: foo
      value: 1
   ```
   </SampleCode>
</Example>

<Example>
   <Description>
      Grab `foo` from `foo.bar` and...
   </Description>
   <SampleCode>
      ```yaml
      processors:
         - set:
            field: foo.bar
            value: "{{bar}}"

      source:
         foo.bar: "BAR"
      ```
   </SampleCode>
   <Description>
   ...copy it into `bar`
   </Description>
   <SampleCode>
      ```yaml
      source:
         foo:
            - bar:
               value: "BAR"
      ```
   </SampleCode>
</Example>

* It is possible to access array fields with the dot notation.

<Example>
   <Description>
      The `foo` field contains `bar` with 3 elements:
   </Description>
   <SampleCode>
      ```yaml
      - pipeline:
         processors:
            - set:
               field: foo
               value: "{{bar.1}}"

      - source:
        foo: ["A", "B", "C"]
      ```
   </SampleCode>
   <Description>
      The _1st_ is copied to _bar_:
   </Description>
   <SampleCode>
      ```yaml
      source:
         - input_field: ["A", "B", "C"]
         - bar: "A"
      ```
   </SampleCode>
</Example>

* The contents of a field including complex values such as arrays or objects can be copied to another field using `copy_from`:

<Example>
   <Description>
      The values in `foo`...
   </Description>
   <SampleCode>
      ```yaml
      processors:
         - set:
            field: bar
            copy_from: foo
      ...

      source:
         foo: ["A", "B"]
      ```
   </SampleCode>
   <Description>
      ...are copied into `bar`:
   </Description>
   <SampleCode>
      ```yaml
      source:
         foo: ["A", "B"]
         bar: ["A", "B"]
      ```
   </SampleCode>
</Example>
