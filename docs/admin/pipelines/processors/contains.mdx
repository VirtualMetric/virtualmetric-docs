# Contains

## Synopsis

```yaml {2}
contains:
   - field: <ident>
   - description: <string>
   - if: <script>
   - ignore_failure: <logical>
   - ignore_missing: <logical>
   - list: <strings...>
   - list_field: <ident>
   - on_failure: <processors...>
   - on_success: <processors...>
   - tag: <string>
```

## Description

**Functionality**: Checks the existence of a value in a list of values.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y|N/A|Field that contains the value to check|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`ignore_missing`|N|`false`|If `true` and `field` does not exist or is `null`, exit quietly without making any modifications|
|`list`|N|-|List of values to check|
|`list_field`|N|-|Field containing the list of values to check|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`on_success`|N|-|See [Handling Success](../handling-success.mdx)|
|`tag`|N|-|Identifier|

## Examples

<ExampleTable>
   <Description>
   Values that exist are returned:
   </Description>
   <SampleCode>
   ```yaml
   contains:
      - field: data
      - list:
         - "foo"
         - "baz"
   data:
      - foo
      - bar
      - baz
      - qux
   ```
   </SampleCode>

   <Description>
   Values that do not exist are skipped:
   </Description>
   <SampleCode>
   ```yaml
   contains:
      field: data
      list: "qux"

   data:
      - foo
      - bar
      - baz
   ```
   </SampleCode>
</ExampleTable>
