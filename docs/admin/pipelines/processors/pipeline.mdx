# Pipeline

## Synopsis

```yaml {2}
pipeline:
  - name: <ident>
  - description: <text>
  - if: <script>
  - ignore_failure: <boolean>
  - ignore_missing_pipeline: <boolean>
  - on_failure: <processor[]>
  - on_success: <processor[]>
  - tag: <string>
```

## Description

**Functionality**: Executes another pipeline.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`name`|Y||Name of the pipeline to be executed|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`ignore_missing_pipeline`|N|`false`|Flag to fail at a missing pipeline|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`on_success`|N|-|See [Handling Success](../handling-success.mdx)|
|`tag`|N|-|Identifier|

## Examples

Run pipeline _bar_ after you are finished running _foo.processor-1_:

```yaml
pipelines:
  - foo:
    processors:
    - processor-1:
      field: "A"
      pipeline: bar
  - bar:
    processors:
    ...
```
