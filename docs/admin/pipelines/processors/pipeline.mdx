# Pipeline

<Synopsis>
Executes another pipeline.
</Synopsis>

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`name`|Y|N/A|The name of the pipeline to be executed|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`ignore_missing_pipeline`|N|`false`|Flag to fail at a missing pipeline|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`on_success`|N|-|See [Handling Success](../handling-success.mdx)|
|`tag`|N|-|Identifier|

## Examples

Run pipeline **bar** after you have finished **foo.processor-1**:

```json
{
   "pipelines": [
      {
         "id": "foo",
         "processors": [
            "processor-1" {
               "field": "A",
               "pipeline": "bar"
            }
         ]
      },
      {
         "id": "bar",
         "processors": [
            // etc.
         ]
      }

   ]
}
```
