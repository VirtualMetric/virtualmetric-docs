# Normalize

## Synopsis
```yaml
normalize:
   - description: <string>
   - if: <script>
   - ignore_failure: <logical>
   - on_failure: <processors...>
   - on_success: <processors...>
   - source_format: <ident>
   - tag: <string>
   - target_format: <ident>
```

## Description

**Functionality**: Converts the source type fields to the target type fields such as [**ECS**](/docs/appendix/log-types/ecs.mdx) to [**CIM**](/docs/appendix/log-types/cim.mdx) or [**ASIM**](/docs/appendix/log-types/asim.mdx), or vice versa.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`on_success`|N|-|See [Handling Success](../handling-success.mdx)|
|`source_format`|N|-|Field containing the source type|
|`tag`|N|-|Identifier|
|`target_format`|N|-|Field to assign converted type|

## Examples

Normalize the data in field `foo` by converting it to the **ASIM** format:

```yaml
- normalize:
   source: foo
   target_format: ASIM
```
