# Dot Expander

<Synopsis>
Expands a field containing dots into an object field. Makes fields with dots in their names accessible to other processors.
</Synopsis>

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y|N/foo|Field to be expanded. If set to `*`, all top-level fields will be expanded|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`on_success`|N|-|See [Handling Success](../handling-success.mdx)|
|`path`|N|-|Field containing the field to expand. Only required when `field` is part of another object field since only leaf level fields can be processed|
|`tag`|N|-|Identifier|

## Examples

* If the expanded name is distinct, the part after the dot becomes nested:

<Example>
   <Description>
      **Input**
   </Description>
   <SampleCode>
      ```json
      {
         "foo.bar": "v"
      }
      ```
   </SampleCode>
   <Description>
      **Spec**
   </Description>
   <SampleCode>
      ```json
      {
         "dot_expander": {
            "field": "foo.bar"
         }
      }
      ```
   </SampleCode>
   <Description>
      **Output**
   </Description>
   <SampleCode>
      ```json
      {
         "foo": {
            "bar": "v"
         }
      }
      ```
   </SampleCode>
</Example>

* If the dotted field clashes with a nested one, it is merged with it:

<Example>
   <Description>
      **Input**
   </Description>
   <SampleCode>
      ```json
      {
         "foo.bar": "v1",
         "foo": {
            "bar": "v2"
         }
      }
      ```
   </SampleCode>
   <Description>
      **Spec**
   </Description>
   <SampleCode>
      ```json
      {
         "dot_expander": {
            "field": "foo.bar"
         }
      }
      ```
   </SampleCode>
   <Description>
      **Output**
   </Description>
   <SampleCode>
      ```json
      {
         "foo": {
            "bar": ["v1", "v2"]
         }
      }
      ```
   </SampleCode>
</Example>

* Using `override` causes the expanded field to be overridden by the nested one

<Example>
   <Description>
      **Input**
   </Description>
   <SampleCode>
      ```json
      {
         "foo.bar": "v1",
         "foo": {
            "bar": "v2"
         }
      }
      ```
   </SampleCode>
   <Description>
      **Spec**
   </Description>
   <SampleCode>
      ```json
      {
         "dot_expander": {
            "field": "foo.bar",
            "override": true
         }
      }
      ```
   </SampleCode>
   <Description>
      **Output**
   </Description>
   <SampleCode>
      ```json
      {
         "foo": {
            "bar": "v2"
         }
      }
      ```
   </SampleCode>
</Example>

* Using `*` expands all top-level fields

<Example>
   <Description>
      **Input**
   </Description>
   <SampleCode>
      ```json
      {
         "foo.bar": "v1",
         "baz.qux": "v2"
      }
      ```
   </SampleCode>
   <Description>
      **Spec**
   </Description>
   <SampleCode>
      ```json
      {
         "dot_expander": {
            "field": "*"
         }
      }
      ```
   </SampleCode>
   <Description>
      **Output**
   </Description>
   <SampleCode>
      ```json
      {
         "foo": {
            "bar": "v1"
         },
         "baz": {
            "qux": "v2"
         }
      }
      ```
   </SampleCode>
</Example>

* If the dotted field is nested within a field, use `path` to navigate down

<Example>
   <Description>
      **Input**
   </Description>
   <SampleCode>
      ```json
      {
         "foo": {
            "bar.baz": "v1",
            "bar.qux": "v2"
         }
      }
      ```
   </SampleCode>
   <Description>
      **Spec**
   </Description>
   <SampleCode>
      ```json
      {
         "dot_expander": {
            "path": "foo",
            "field": "*"
         }
      }
      ```
   </SampleCode>
   <Description>
      **Output**
   </Description>
   <SampleCode>
      ```json
      {
         "foo": {
            "bar": {
               "baz": "v1",
               "qux": "v2"
            }
         }
      }
      ```
   </SampleCode>
</Example>

* If there is a potential for a clash between an expanded field and another one at the same level, use `rename` first

<Example>
   <Description>
      **Input**
   </Description>
   <SampleCode>
      ```json
      {
         "foo": "v1",
         "foo.bar": "v2"
      }
      ```
   </SampleCode>
   <Description>
      **Spec**
   </Description>
   <SampleCode>
      ```json
      {
         "processors": [
            {
               "rename": {
                  "field": "foo",
                  "target_field": "foo.bar"
               }
            },
            {
               "dot_expander": {
                  "path": "foo",
                  "field": "*"
               }
            }
         ]
      }
      ```
   </SampleCode>
   <Description>
      **Output**
   </Description>
   <SampleCode>
      ```code
      {
         "foo": {
            "bar": ["v1", "v2"]
         }
      }
      ```
   </SampleCode>
</Example>
