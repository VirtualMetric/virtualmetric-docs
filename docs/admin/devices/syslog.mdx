# Syslog

## Description

Represents _syslog_, a logging format offering the flexibility to filter, analyze, and store logs in a centralized location for monitoring, troubleshooting, and compliance purposes.

## Common Settings

**Transport** -\
<tab-ln/>**UDP** or **TCP** protocols\
<tab-ln/>Default ports:\
<tab-ln indent="4"/>`514` for either\
**Encryption** -\
<tab-ln/>For secure transport, use with **TLS**

## Content

Provides the following information:

**Timestamp** -\
<tab-ln/>Date and time of _event_\
<tab-ln/>Format: `YYYY-MM-DD HH:MM:SS`\
**Hostname / IP Address** -\
<tab-ln/>System where _event_ originated\
**Facility** -\
<tab-ln/>Type of program or service generating the log message.\
<tab-ln/>Examples:\
<tab-ln indent="4"/>**auth**: Authentication\
<tab-ln indent="4"/>**cron**: Scheduled job\
<tab-ln indent="4"/>**daemon**: System\
<tab-ln indent="4"/>**kern**: Kernel\
<tab-ln indent="4"/>**mail**: E-mail\
**Severity** -\
<tab-ln/>Urgency of the message\
<tab-ln/>Standard levels:\
<tab-ln indent="4"/>**0**: Emergency. System is unusable\
<tab-ln indent="4"/>**1**: Alert. Immediate action required\
<tab-ln indent="4"/>**2**: Critical condition\
<tab-ln indent="4"/>**3**: Error conditions\
<tab-ln indent="4"/>**4**: Warning conditions\
<tab-ln indent="4"/>**5**: Notice. Normal but significant conditions\
<tab-ln indent="4"/>**6**: Informational messages\
<tab-ln indent="4"/>**7**: Debugging messages\
**Message** -\
<tab-ln/>Actual log content. Provides details about the event or action, such as\
<tab-ln/>errors, process activities, or custom application messages\
**Process Information** -\
<tab-ln/>Name of the process or application generating the log.\
<tab-ln/>May include **PID** (Process ID)\
**Remote Logging** (Optional) -\
<tab-ln/>If _syslog_ messages are forwarded to a central server, the logs may include:\
<tab-ln indent="4"/>- The source IP of the sender\
<tab-ln indent="4"/>- Network-related metadata\
**Priority** -\
<tab-ln/>Combination of facility and severity encoded into a single numeric value\
**Metadata** -\
<tab-ln/>Protocol and port number. Used when messages are transported over a network

      address: "0.0.0.0"
      port: 514
