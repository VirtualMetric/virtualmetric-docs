# Syslog

## Synopsis

```yaml
devices:
  - id: <numeric>
    name: <ip-address>
    description: <string>
    type: http
    status: <logical>
    properties:
      network: <string>
      address: <string>
      port: <numeric>
      tls: <logical>
      cert_name: <string>
      key_name: <string>
      reuse: <logical>
      flush_interval: <numeric>
      queue:
        interval: <numeric>
      batch_size: <numeric>
```

## Description

Represents the _syslog_ format which offers the flexibility to filter, analyze, and store logs in a centralized location for monitoring, troubleshooting, and compliance purposes.

## Parameters

|Field|Subfield|Required|Default|Description|
|---|---|---|---|---|
|`id`|-|Y|||
|`name`|-|Y|`127.0.0.1`||
|`description`|-|N|-|Explanatory note|
|`type`|-|Y|`syslog`||
|`status`|-|Y|`false`||
|`properties`|`network`|Y|`"udp"`||
||`address`|Y|`"0.0.0.0"`||
||`port`|Y|`14514`||
||`tls`||`false`||
||`cert_name`||`""`||
||`key_name`||`""`||
||`reuse`||`false`||
||`flush_interval`||`1`||
||`queue.interval`||`1`||
||`batch_size`||`1000`||
