# Syslog

## Description

Represents _syslog_, a logging format offering the flexibility to filter, analyze, and store logs in a centralized location for monitoring, troubleshooting, and compliance purposes.

## Common Settings

**Encryption** -\
<tab-ln/>For secure transport, use with **TLS**\
**Protocol** -\
<tab-ln/>**UDP** or **TCP**\
**Port** -\
<tab-ln/>**514**\
**Encryption** -\
<tab-ln/>For secure transport, use with **TLS**

## Content

Provides the following information items:

**Timestamp** -\
<tab-ln/>Date and time of _event_\
<tab-ln/>Format: YYYY-MM-DD HH:MM:SS\
**Hostname / IP Address** -\
<tab-ln/>System where _event_ originated\
**Facility** -\
<tab-ln/>Type of program or service generating the log message\
<tab-ln/>Examples:\
<tab-ln/><tab-ln/>**auth**: Authentication\
<tab-ln/><tab-ln/>**cron**: Scheduled job\
<tab-ln/><tab-ln/>**daemon**: System\
<tab-ln/><tab-ln/>**kern**: Kernel\
<tab-ln/><tab-ln/>**mail**: E-mail\
**Severity** -\
<tab-ln/>Urgency of the message\
<tab-ln/>Standard levels:\
<tab-ln/><tab-ln/>**0**: Emergency. System is unusable\
<tab-ln/><tab-ln/>**1**: Alert. Immediate action required\
<tab-ln/><tab-ln/>**2**: Critical condition\
<tab-ln/><tab-ln/>**3**: Error conditions\
<tab-ln/><tab-ln/>**4**: Warning conditions\
<tab-ln/><tab-ln/>**5**: Notice. Normal but significant conditions\
<tab-ln/><tab-ln/>**6**: Informational messages\
<tab-ln/><tab-ln/>**7**: Debugging messages\
**Message** -\
<tab-ln/>Actual log content. Provides details about the event or action, such as\
<tab-ln/>errors, process activities, or custom application messages\
**Process Information** -\
<tab-ln/>Name of the process or application generating the log.\
<tab-ln/>May include **PID** (Process ID)\
**Remote Logging** (Optional) -\
<tab-ln/>If syslog messages are forwarded to a central server, the logs may include:\
<tab-ln/><tab-ln/>- The source IP of the sender\
<tab-ln/><tab-ln/>- Network-related metadata\
**Priority** -\
<tab-ln/>Combination of facility and severity encoded into a single numeric value\
**Metadata** -\
<tab-ln/>Protocol and port number. Used when messages are transported over a network

## Typical Use

<details>
<summary>Basic `syslog.yaml` configuration</summary>
```yaml
devices:
   id: {0..9}+
   name: "<some-name>"
   type: syslog
   properties:
      address: "0.0.0.0"
      port: 514
```
</details>

With the above YAML settings, this device can do the following:

- Listen to **Syslog** over a regular **UDP** connection
- Listen to **Syslog** over a regular **TCP** connection launched by a pre-processor pipeline
- Launch a **Syslog** over **TLS**; requires configuring how to introduce a server certificate
- Collect data with a **TCP** line parser
