# SNMP Trap

## Synopsis

```yaml {5}
devices:
  - id: <numeric>
    name: <ip-numeric>
    description: <text>
    type: snmptrap
    status: <logical>
    properties:
      address: <ip-string>
      port: <numeric>
      community: <string>
      username: <string>
      password: <string>
      authentication:
        status: <logical>
        protocol: <numeric>
        password: <string>
      privacy:
        protocol: <numeric>
        passphrase: <string>
```

## Description

## Parameters

Basic configuration.

|Field|Required|Default|Description|
|---|---|---|---|
|`id`|Y|N/A|Device identifier|
|`name`|Y|N/A|Address used to refer to the listener, e.g. `127.0.0.1`|
|`description`|N|-|Explanatory note|
|`type`|Y|N/A|Type of device. Here, `smmptrap`|
|`status`|N|`true`|Enable device|

### `properties:`

Device-specific settings.

|Field|Required|Default|Description|
|---|---|---|---|
|`address`|N|`"0.0.0.0"`|Listening address|
|`port`|Y|N/A|Listening port|
|`community`|N|`""`|Community string. Required if authentication v2 is used|
|`username`|N|`""`|Authentication username|
|`password`|N|`""`|Authentication password|

### `authentication:`

|Field|Required|Default|Description|
|---|---|---|---|
|`status`|N|`false`|Enable authentication v3. Required if authentication v3 is used|
|`protocol`|N|`1`|Authentication protocol. Required if authentication v3 is used|
|`password`|N|`""`|Authentication password. Required if authentication v3 is used|

|---|---|---|---|
