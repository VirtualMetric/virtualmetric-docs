The protocol offers the following features:

* **Statelessness**: Each request is independent, with no inherent connection between them
* **Caching**: Mechanisms like `ETag`, `Cache-Control`, and `Expires` for optimizing resource delivery
* **Compression**: Encoding like `gzip` or `deflate` for reducing payload size
* **Persistence**: With `HTTP/1.1` and later, connections can be reused for multiple requests
* **Security** (HTTPS): Uses SSL/TLS to encrypt HTTP traffic, providing confidentiality and integrity

This structure allows HTTP to support a wide range of applications, from simple websites to complex APIs and distributed systems.

## Protocol Content

**Request Line** (for HTTP requests) -\
<tab-ln/>`GET`, `POST`, `PUT`, `DELETE`, `HEAD`, `OPTIONS`, `PATCH`\
**URI**  (Uniform Resource Identifier) -\
<tab-ln/>Resource being requested (`HTTP/1.1` or `HTTP/2`)\
**Status Line** (for HTTP responses) -\
<tab-ln/>HTTP version used in the response\
<tab-ln/>Status Code: 3-digit code indicating the result of the request\
<tab-ln/><tab-ln/>`200 OK`: Successful request\
<tab-ln/><tab-ln/>`301 Moved Permanently`: Resource moved\
<tab-ln/><tab-ln/>`404 Not Found`: Resource not found\
<tab-ln/><tab-ln/>`500 Internal Server Error`: Server error\
<tab-ln/>Reason phrase: textual description of the status code\
**Headers** -\
<tab-ln/>Key-value pairs that provide metadata about the request or response\
<tab-ln/>Request Headers:\
<tab-ln/><tab-ln/>`Host`: server's hostname\
<tab-ln/><tab-ln/>`User-Agent`: client making the request\
<tab-ln/><tab-ln/>`Accept`: acceptable content types (e.g. `text/html`)\
<tab-ln/><tab-ln/>`Authorization`: credentials for authentication\
<tab-ln/>Response Headers:\
<tab-ln/><tab-ln/>`Content-Type`: type of content returned (e.g. `application/json`)\
<tab-ln/><tab-ln/>`Content-Length`: size of the response body\
<tab-ln/><tab-ln/>`Set-Cookie`: cookie in the client\
<tab-ln/><tab-ln/>`Server`: info about the server handling the request\
**Body** -\
<tab-ln/>Data being sent or received\
<tab-ln/>Request Body:\
<tab-ln/><tab-ln/>Used with methods like `POST` or `PUT` to send data\
<tab-ln/>Response Body:\
<tab-ln/><tab-ln/>Contains the content of the response\
**Methods** (Request Type) -\
<tab-ln/>Desired operation on the resource:\
<tab-ln/><tab-ln/>`GET`: Retrieve data from the server\
<tab-ln/><tab-ln/>`POST`: Submit data to the server\
<tab-ln/><tab-ln/>`PUT`: Update a resource\
<tab-ln/><tab-ln/>`DELETE`: Remove a resource\
**Status Codes** (Response Type) -\
<tab-ln/>**1xx (Informational)**: Request received, continuing process\
<tab-ln/>**2xx (Success)**: Request successfully received, understood, and accepted\
<tab-ln/>**3xx (Redirection)**: Further action needed to complete the request\
<tab-ln/>**4xx (Client Error)**: Request contains errors (e.g., bad syntax, unauthorized)\
<tab-ln/>**5xx (Server Error)**: Server failed to fulfill a valid request\
**Cookies** -\
<tab-ln/>Key-value pairs sent by the server to the client to store stateful information.\
<tab-ln/>Included in subsequent requests to the server
