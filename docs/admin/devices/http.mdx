## Content

Provides the following information:

**Request Line** (For HTTP Requests) -\
<tab-ln/>Common methods:\
<tab-ln indent="4"/>`GET`, `POST`, `PUT`, `DELETE`, `HEAD`, `OPTIONS`, `PATCH`\
**URI**  (Uniform Resource Identifier) -\
<tab-ln/>The resource being requested\
<tab-ln/>Version: `HTTP/1.1` or `HTTP/2`\
**Status Line** (For HTTP Responses) -\
<tab-ln/>HTTP Version: version used in the response\
<tab-ln/>Status Code: 3-digit code indicating the result of the request\
<tab-ln/>Common status codes:\
<tab-ln indent="4"/>`200 OK`: Successful request\
<tab-ln indent="4"/>`301 Moved Permanently`: Resource moved\
<tab-ln indent="4"/>`404 Not Found`: Resource not found\
<tab-ln indent="4"/>`500 Internal Server Error`: Server error\
<tab-ln/>Reason phrase: textual description of the status code\
**Headers** -\
<tab-ln/>Key-value pairs that provide metadata about the request or response\
<tab-ln/>Request Headers:\
<tab-ln indent="4"/>`Host`: server's hostname\
<tab-ln indent="4"/>`User-Agent`: client making the request\
<tab-ln indent="4"/>`Accept`: acceptable content types (e.g. `text/html`)\
<tab-ln indent="4"/>`Authorization`: credentials for authentication\
<tab-ln/>Response Headers:\
<tab-ln indent="4"/>`Content-Type`: type of content returned (e.g. `application/json`)\
<tab-ln indent="4"/>`Content-Length`: size of the response body\
<tab-ln indent="4"/>`Set-Cookie`: cookie in the client\
<tab-ln indent="4"/>`Server`: info about the server handling the request\
**Body** -\
<tab-ln/>The data being sent or received\
<tab-ln/>Request Body:\
<tab-ln indent="4"/>Used with methods like `POST` or `PUT` to send data\
<tab-ln/>Response Body:\
<tab-ln indent="4"/>Contains the content of the response\
**Methods** (Request Type) -\
<tab-ln/>Defines the desired operation on the resource\
<tab-ln/>Examples:\
<tab-ln indent="4"/>`GET`: Retrieve data from the server\
<tab-ln indent="4"/>`POST`: Submit data to the server\
<tab-ln indent="4"/>`PUT`: Update a resource\
<tab-ln indent="4"/>`DELETE`: Remove a resource\
**Status Codes** (Response Type) -\
<tab-ln/>Categorized into five classes:\
<tab-ln indent="4"/>**1xx (Informational)**: Request received, continuing process\
<tab-ln indent="4"/>**2xx (Success)**: Request successfully received, understood, and accepted\
<tab-ln indent="4"/>**3xx (Redirection)**: Further action needed to complete the request\
<tab-ln indent="4"/>**4xx (Client Error)**: Request contains errors (e.g., bad syntax, unauthorized)\
<tab-ln indent="4"/>**5xx (Server Error)**: Server failed to fulfill a valid request\
**Cookies** -\
<tab-ln/>Key-value pairs sent by the server to the client to store stateful information\
<tab-ln/>Included in subsequent requests to the server

### Protocol Features

**Statelessness** -\
<tab-ln/>Each request is independent, with no inherent connection between them\
**Caching** -\
<tab-ln/>Mechanisms like `ETag`, `Cache-Control`, and `Expires` for optimizing\
<tab-ln/>resource delivery\
**Compression** -\
<tab-ln/>Encoding like `gzip` or `deflate` for reducing payload size\
**Persistence** -\
<tab-ln/>With `HTTP/1.1` and later, connections can be reused for multiple requests\
**Security** (HTTPS)-\
<tab-ln/>Uses SSL/TLS to encrypt HTTP traffic, providing confidentiality and integrity.

This structure allows HTTP to support a wide range of applications, from simple websites to complex APIs and distributed systems.
