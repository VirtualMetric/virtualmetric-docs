# Rabbitmq

## Synopsis

```yaml {5}
devices:
  - id: <numeric>
    name: <ip-numeric>
    description: <text>
    type: rabbitmq
    status: <logical>
    properties:
      address: <ip-string>
      port: <numeric>
      authentication: <enum-string>
      username: <string>
      password: <string>
      exchange:
        name: <string>
        type: <enum-string>
      queue:
        name: <string>
        key: <string>
      tls:
        status: <logical>
        cert_name: <string>
        key_name: <string>
```

## Description

Represents the listener that deliver data from the RabbitMQ protocol.

## Parameters

Basic configuration.

|Field|Required|Default|Description|
|---|---|---|---|
|`id`|Y|N/A|Device identifier|
|`name`|Y|N/A|Address used to refer to the listener, e.g. `127.0.0.1`|
|`description`|N|-|Explanatory note|
|`type`|Y|N/A|Type of device. Here, `rabbitmq`|
|`status`|N|`true`|Enable device|

### `properties:`

Device-specific settings.

|Field|Required|Default|Description|
|---|---|---|---|
|`address`|N|`"0.0.0.0"`|Listening address|
|`port`|N|`0`|Listening port|
|`authentication`|N|`""`|Authentication type. Valid options: `"plain"`, `"amqplain"`|
|`username`|N|`""`|Required if authentication is used|
|`password`|N|`""`|Required if authentication is used|

### `exchange:`

|Field|Required|Default|Description|
|---|---|---|---|
|`name`|N|`""`|Exchange name|
|`type`|N|`""`|Exchange type. Valid options: `"direct"`, `"fanout"`, `"topic"`, `"x-custom"`|

### `queue:`

|Field|Required|Default|Description|
|---|---|---|---|
|`name`|Y|N/A|Queue name|
|`key`|Y|N/A|Queue key|

### `tls:`

TLS (Transport Security Layer) settings.

|Field|Required|Default|Description|
|---|---|---|---|
|`status`|N|`false`|Enable TLS|
|`cert_name`|N|`"cert.pem"`|TLS certificate file path|
|`key_name`|N|`"key.pem"`|TLS key file path|
