# EStreamer

## Synopsis

```yaml
devices:
  - id: <numeric>
    name: <ip-numeric>
    description: <text>
    type: estreamer
    status: <logical>
    properties:
      address: <ip-string>
      port: <numeric>
      tls:
        non_secure: <logical>
      cert_name: <string>
      key_name: <string>
      reuse: <logical>
      flush_interval: <numeric>
      queue:
        interval: <numeric>
      batch_size: <numeric>
      inputs:
        - id: <numeric>
        name: <string>
        status: <logical>
        ...
```

## Description

Represents that listener that delivers data from **eStreamer**.

The protocol refers to event data collected from Cisco Firepower devices, including Firepower Threat Defense (FTD) and Firepower Management Center (FMC). It is transmitted using the eStreamer protocol, part of Cisco's _Event Streamer API_. The data provides detailed security event information that can be used for monitoring, analysis, and threat detection.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`id`|Y|N/A|Device identifier|
|`name`|Y|N/A|Address of the listener, e.g. `127.0.0.1`|
|`description`|N|-|Explanatory note|
|`type`|Y|N/A|Type of device. Should be `estreamer`|
|`status`|N|`true`|Enable device|

### `properties:`

|Field|Required|Default|Description|
|---|---|---|---|
|`address`|N|`"0.0.0.0"`|Listening address|
|`port`|N|`8302`|Listening port|
|`cert_name`|N|`""`|TLS certificate path|
|`key_name`|N|`""`|TLS key path|
|`reuse`|N|`true`|Enable reuse of same address and port|
|`flush_interval`|N|`1`|Ingester flush interval in sec|
|`batch_size`|N|`1000`|Ingester batch size|

### `inputs:`

|Field|Required|Default|Description|
|---|---|---|---|
|`id`|Y|N/A|Event id|
|`name`|N|-|Event name|
|`status`|N|`true`|Enable event|

### `queue:`

|Field|Required|Default|Description|
|---|---|---|---|
|`interval`|N|`1`|Ingester de-queue interval in sec|

### `tls:`

|Field|Required|Default|Description|
|---|---|---|---|
|`non_secure`|N|`false`|Use to enable non-secure TLS|
