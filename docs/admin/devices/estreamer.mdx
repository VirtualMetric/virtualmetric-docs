# EStreamer

## Synopsis

```yaml
devices:
  - id: <numeric>
    name: <ip-numeric>
    description: <text>
    type: estreamer
    status: <logical>
    properties:
      address: <ip-string>
      port: 8302
      tls:
        non_secure: <logical>
      cert_name: <string>
      key_name: <string>
      reuse: <logical>
      flush_interval: <numeric>
      queue:
        interval: <numeric>
      batch_size: <numeric>
      inputs:
        - id: <numeric>
        name: <string>
        status: <logical>
        ...
```

## Description

**eStreamer** data refers to event data collected from Cisco Firepower devices, including Firepower Threat Defense (FTD) and Firepower Management Center (FMC). It is transmitted using the eStreamer protocol, part of Cisco's Event Streamer API. The data provides detailed security event information that organizations can use for monitoring, analysis, and threat detection.

### Protocol Content

<dl>
  <dt>**Intrusion Events**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>
    Alerts for detected intrusions and their associated metadata, including details such as event timestamps, signatures, priorities, and classifications
    </dd>

  <dt>**Connection Events**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>
    Information about network traffic connections, including source/destination IPs, ports, protocols, and byte counts. Flags indicating connection status (e.g. `allowed`, `blocked`)
    </dd>

  <dt>**Security Intelligence Events**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Logs of traffic that matches Cisco's Security Intelligence blacklists (e.g. malicious IPs, URLs, or domains)</dd>

  <dt>**Malware Events**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Detection events for malware or file-based threats, including file hashes and types</dd>

  <dt>**Threat Events**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Identified threat-related data, including severity and remediation information</dd>

  <dt>**Host and User Details**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Associated hostnames, user identities (via integration with identity providers), and network context</dd>

  <dt>**File and Malware Data**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Metadata for inspected files, including file name, type, size, and hash values (e.g. `MD5`, `SHA-256`)</dd>

  <dt>**System and Audit Logs**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>System health and configuration changes within the Firepower devices</dd>

  <dt>**Policy Details**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Correlation with applied access control or intrusion prevention policies</dd>

  <dt>**Application Details**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Application detection data, identifying traffic based on application types (e.g. **Facebook**, **Dropbox**)</dd>

  <dt>**URL Filtering Events**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Logs of URL requests matched against filtering policies (e.g. blocked or allowed URLs)</dd>

  <dt>**VPN and SSL Event Data**</dt>
    <hr style={{margin: "0px"}}/>
    <dd>Events related to VPN and encrypted traffic inspections</dd>
</dl>
