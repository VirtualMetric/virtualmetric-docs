# EStreamer

## Synopsis

```ssml {5}
- id: <numeric>
  name: <ip-numeric>
  description: <text>
  type: estreamer
  status: <logical>
  properties:
    address: <ip-string>
    port: <numeric>
    tls:
      non_secure: <logical>
    cert_name: <string>
    key_name: <string>
    flush_interval: <numeric>
    queue:
      interval: <numeric>
    batch_size: <numeric>
    inputs:
      - id: <numeric>
      name: <string>
      status: <logical>
      ...
```

## Description

Represents the listener that delivers data through **eStreamer**.

The protocol refers to event data collected from Cisco Firepower devices, including Firepower Threat Defense (FTD) and Firepower Management Center (FMC). It is transmitted using the eStreamer protocol, part of Cisco's _Event Streamer API_. The data provides detailed security event information that can be used for monitoring, analysis, and threat detection.

## Parameters

<BasicConfig />

### `properties:`

Device-specific settings.

|Field|Required|Default|Description|
|---|---|---|---|
|`address`|N|`"0.0.0.0"`|Listening address|
|`port`|N|`8302`|Listening port|
|`cert_name`|N|`""`|TLS certificate path|
|`key_name`|N|`""`|TLS key path|
|`flush_interval`|N|`1`|Ingester flush interval in sec|
|`batch_size`|N|`1000`|Ingester batch size|

### `inputs:`

Event settings.

|Field|Required|Default|Description|
|---|---|---|---|
|`id`|Y|N/A|Event id|
|`status`|N|`true`|Enable event|

### `queue:`

Ingester queue settings.

|Field|Required|Default|Description|
|---|---|---|---|
|`interval`|N|`1`|Ingester de-queue interval in sec|

### `tls:`

Transport Layer Security settings.

|Field|Required|Default|Description|
|---|---|---|---|
|`non_secure`|N|`false`|Enable non-secure TLS|
