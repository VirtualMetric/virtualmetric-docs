This guide walks you through configuring your first device in Director, using a syslog server as an example.

## Configuration Structure

Director searches recursively through the `config` directory for configuration files with either `.yaml` or `.yml` extensions. You can organize your configurations in several ways:

```bash
<director_root>/config/syslog.yaml                     # Using .yaml extension
<director_root>/config/devices/syslog.yml              # Using .yml extension
<director_root>/config/device/listeners/syslog.yaml    # Categorized structure
```

Choose the organization that best fits your needs - Director will find the configurations regardless of subfolder structure.

## Basic UDP Syslog Setup

### 1. Create Configuration File
Create a syslog configuration in your preferred location (e.g., `config/devices/syslog.yaml`):

```yaml
- id: 1                   # Unique identifier for this device
  name: syslog_udp        # Descriptive name
  type: syslog            # Device type
  properties:             
    protocol: udp         # Using UDP protocol
    address: 0.0.0.0      # Listen on all interfaces
    port: 514             # Standard syslog port
```

### 2. Test Configuration
You can test using VirtualMetric Generator (cross-platform) or traditional tools:

**Using VirtualMetric Generator**
```bash
vmetric-generator -mode=syslog -address=127.0.0.1:514 -message="Test Message"     # Linux
vmetric-generator.exe -mode=syslog -address=127.0.0.1:514 -message="Test Message" # Windows
```

**Using System Logger (Unix/Linux)**
```bash
logger -n 127.0.0.1 -P 514 "Test message"
```

## Secure TCP Syslog Setup

### 1. Prepare TLS Certificates

**Using VirtualMetric Director (Cross-Platform)**
```bash
vmetric-director -pfx2pem=./mycert.pfx     # Linux
vmetric-director.exe -pfx2pem=./mycert.pfx # Windows
# Generates: key.pem and cert.pem
```

**Using OpenSSL (Unix/Linux)**
```bash
openssl genrsa -out server.key 2048
openssl req -new -x509 -key server.key -out server.crt -days 365
```

### 2. Create Secure Configuration
Update or create your configuration file (e.g., `config/devices/secure_syslog.yaml`):

```yaml
- id: 2
  name: syslog_secure
  type: syslog
  properties:
    protocol: tcp
    address: 0.0.0.0
    port: 6514
    tls:
      status: true
      cert_name: cert.pem    # Generated from pfx2pem
      key_name: key.pem      # Generated from pfx2pem
    framing: octet
    max_connections: 1000
    timeout: 300
    batch_size: 5000
    queue:
      interval: 2
```

### 3. Place TLS Certificates
Copy the certificate files to your Director root directory:
```bash
cp cert.pem key.pem <director_root>/
```

### 4. Test Secure Configuration
Test the secure connection:
```bash
echo "Test secure message" | openssl s_client -connect 127.0.0.1:6514
```

## Performance Tuning

For high-volume environments, add these settings to your configuration:

```yaml
    reuse: true                # Enable socket reuse
    workers: 4                 # Use 4 worker processes
    buffer_size: 32768         # 32KB buffer size
    flush_interval: 5          # Flush every 5 seconds
```
