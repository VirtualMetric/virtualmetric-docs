# Kafka

## Synopsis

```yaml
devices:
  - id: <numeric>
    name: <ip-numeric>
    description: <text>
    type: kafka
    status: <logical>
    properties:
      address: <ip-string>
      port: <numeric>
      username: <string>
      password: <string>
      partitions: <numeric>
      group: <string>
      balancer: <enum-string>
      topic: <string>
      algorithm: <enum-string>
      tls:
        status: <logical>
        cert_name: <string>
        key_name: <string>
```

## Description

Represents the listener that delivers data streamed with the **Kafta** protocol.

Kafka is a distributed event streaming platform designed for high-throughput, fault-tolerant, and durable data pipelines. It is ideal for use cases such as real-time analytics, event sourcing, and stream processing, leveraging a publish-subscribe messaging model.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`id`|Y|N/A|Device identifier|
|`name`|Y|N/A|Address of the listener, e.g. `127.0.0.1`|
|`description`|N|-|Explanatory note|
|`type`|Y|N/A|Type of device. Should be `kafka`|
|`status`|N|`true`|Enable device|

### `properties:`

|Field|Required|Default|Description|
|---|---|---|---|
|`address`|N|`"0.0.0.0"`|Listening address|
|`algorithm`|N|`""`|Required if authentication is used. Valid options: `"none"`, `"plain"`, `"scram-sha-256"`, `"scram-sha-512"`|
|`balancer`|N|`"roundrobbin"`|Consumer group balancing algorithm. Valid options: `"roundrobin"`, `"cooperativesticky"`, `"range"`, `"sticky"`|
|`group`|N|`"vmetric"`|Consumer group|
|`partition`|N|`1`||
|`password`|N|`""`|Required if authentication is used|
|`port`|Y|N/A|Listening port|
|`topic`|Y|N/A|Consumer topic|
|`username`|N|`""`|Required if authentication is used|

### `tls:`

|Field|Required|Default|Description|
|---|---|---|---|
|`cert_name`|N|`"cert.pem"`|TLS certificate file path|
|`key_name`|N|`"key.pem"`|TLS key file path|
|`status`|N|`false`|Enable TLS connection|
