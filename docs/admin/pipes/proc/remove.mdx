# Remove

<Synopsis>
Removes existing fields. If a field doesnâ€™t exist, an exception is raised.
</Synopsis>

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y|N/A|List of fields to remove|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_failure`|N|`false`|See [Handling Failures](../handling-failures.mdx)|
|`ignore_missing`|N|`false`|If `true` and `field` does not exist or is `null`, exit quietly without making any modifications|
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`tag`|N|-|Identifier|

## Examples

We have three fields: **foo**, **bar**, and **baz**.

<Example>
   <Description>
      We want to remove **foo**:
   </Description>
   <SampleCode>
      ```js
      {
         "remove": {
            "field": "foo"
         }
      }
      ```
   </SampleCode>
   <Description>
      We want to remove **foo** and **baz**:
   </Description>
   <SampleCode>
      ```js
      {
         "remove": {
            "field": ["foo", "baz"]
         }
      }
      ```
   </SampleCode>
</Example>
