# XML

<span class="theme-doc-version-badge badge badge--primary">Parse</span>

## Synopsis

```yaml {2-3}
xml:
  - field: <ident>
  - target_field: <ident>
  - description: <text>
  - if: <script>
  - add_to_root: <boolean>
  - ignore_failure: <boolean>
  - ignore_missing: <boolean>
  - on_failure: <processor[]>
  - on_success: <processor[]>
  - tag: <string>
```

## Description

**Functionality**: Parses XML-formatted strings into structured maps. The processor converts XML documents into nested map structures, handling complex scenarios like nested elements, attributes, and mixed content.

The XML parsing provides special handling for:
- XML attributes (stored under `_attributes`)
- Mixed content (stored under `_text` or `_content`)
- Repeated elements (converted to arrays)
- Nested structures

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y||Field containing the XML string to parse|
|`target_field`|N||Field to store the parsed XML structure. If omitted and `add_to_root` is false, an error is raised|
|`description`|N|-|Explanatory note|
|`if`|N|-|Conditional expression to determine if processing should occur|
|`add_to_root`|N|`false`|If true, adds parsed XML elements directly to the log entry root|
|`ignore_failure`|N|`false`|Skip processing if an error occurs|
|`ignore_missing`|N|`false`|Skip processing if the source field is missing|
|`on_failure`|N|-|Processors to run if processing fails|
|`on_success`|N|-|Processors to run after successful processing|
|`tag`|N|-|Identifier for logging purposes|

## Parsing Behavior

### Attributes
Attributes are stored in a special `_attributes` map within the element.

### Content Handling
- Text content is stored under `_text` (for elements with attributes)
- Text content is stored under `_content` (for elements without attributes)
- Mixed content preserves both element structure and text

### Repeated Elements
Repeated elements are automatically converted to arrays.

### Basic XML Parsing

### XML with Attributes

<ExampleGrid>
  <CommentCol>

### Complex Nested Structure

<ExampleGrid>
  <CommentCol>
    Parse complex XML with nested elements and attributes...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "xml_data": "<library><category name=\"fiction\"><book id=\"1\"><title>Book1</title><authors><author>Author1</author><author>Author2</author></authors></book></category></library>"
    }
    ```
    ```yaml
    xml:
      - field: xml_data
      - target_field: parsed_data
    ```
  </CodeCol>
  <CommentCol>
    preserves nested structure and attributes:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "parsed_data": {
        "library": {
          "category": {
            "_attributes": {"name": "fiction"},
            "book": {
              "_attributes": {"id": "1"},
              "title": "Book1",
              "authors": {
                "author": ["Author1", "Author2"]
              }
            }
          }
        }
      }
    }
    ```
  </CodeCol>
</ExampleGrid>

:::note
The XML processor handles complex XML structures by converting them into nested maps and arrays, preserving both content and attributes.
:::

:::warning
Invalid XML will cause processing to fail unless `ignore_failure` is set to `true`.
:::
