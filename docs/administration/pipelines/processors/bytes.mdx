# Bytes

## Synopsis

```yaml {2}
bytes:
  - field: <ident>
  - description: <text>
  - if: <script>
  - ignore_missing: <boolean>
  - ignore_failure: <boolean>
  - on_failure: <processor[]>
  - on_success: <processor[]>
  - tag: <string>
  - target_field: <ident>
```

## Description

Converts a string expressing size written in byte units to its value in bytes.

## Parameters

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y||The filed containing the unit values|
|`description`|N|-|Explanatory note|
|`if`|N|-|Condition to run|
|`ignore_missing`|N|`false``|If `true` and `field` does not exist or contains no value, exit quietly without making any modifications|
|`ignore_failure`|N|-||
|`on_failure`|N|-|See [Handling Failures](../handling-failures.mdx)|
|`on_success`|N|-|See [Handling Success](../handling-success.mdx)|
|`tag`|N|-|Identifier|
|`target_field`|N|-|Field to assign the converted value to, if distinct from `field`|

## Details

If the field contains multiple values, all of them are converted.

Allowed units are `B`, `KB`, `MB`, `GB`, `TB`, and `PB`, all case insensitive. Fractional values like "`1.5KB`" are also supported. If the string is not in one of the enumerated formats or exceeds $2^{64}-1$ in value, an error is raised.

## Examples

### Basic Conversion

<ExampleGrid>
  <CommentCol>
    Convert size with byte units...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "file_size": "1.5KB"
    }
    ```
    ```yaml
    bytes:
      - field: file.size
    ```
  </CodeCol>
  <CommentCol>
    to its value in bytes:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "file.size": 1536
    }
    ```
  </CodeCol>
</ExampleGrid>

### Unit Variations

<ExampleGrid>
  <CommentCol>
    The processor handles various unit formats and cases...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "sizes": {
        "a": "1kb",
        "b": "1KB",
        "c": "1 KB",
        "d": "1 Kb"
      }
    }
    ```
    ```yaml
    bytes:
      - field: sizes.a
    bytes:
      - field: sizes.b
    bytes:
      - field: sizes.c
    bytes:
      - field: sizes.d
    ```
  </CodeCol>
  <CommentCol>
    all converting to the same value:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "sizes": {
        "a": 1024,
        "b": 1024,
        "c": 1024,
        "d": 1024
      }
    }
    ```
  </CodeCol>
</ExampleGrid>

### Different Target Field

<ExampleGrid>
  <CommentCol>
    Store the result in a different field...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "raw_size": "1GB"
    }
    ```
    ```yaml
    bytes:
      - field: raw_size
      - target_field: size_in_bytes
    ```
  </CodeCol>
  <CommentCol>
    keeping the original value:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "raw_size": "1GB",
      "size_in_bytes": 1073741824
    }
    ```
  </CodeCol>
</ExampleGrid>

### Error Handling

<ExampleGrid>
  <CommentCol>
    With invalid formats...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "size": "100XB"
    }
    ```
    ```yaml
    bytes:
      - field: size
      - ignore_failure: true
    ```
  </CodeCol>
  <CommentCol>
    errors can be ignored:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "size": "100XB",
      "error": {
        "message": "Invalid byte unit format"
      }
    }
    ```
  </CodeCol>
</ExampleGrid>
