# Creating Targets

To create a new target,

- In the **Home** view, click **Fleet Management**&nbsp;>&nbsp;**Targets**

-or-

- From the hamburger menu on the top left, select **Fleet Management**&nbsp;>&nbsp;**Targets**

  This will take you to the **Targets** view.

- Click the **Add new target** button, then select the desired target type by clicking its card.

  This opens the first tab of the target-creation flow for that type.

## General Settings

- **Name** (Required) - Unique identifier for the target
- **Description** (Optional) - A note describing the purpose of the target
- **Target Status** - Enable/disable target. By default, enabled
- **Post-processing pipeline** (Optional) - Select a pipeline to normalize the output from the dropdown. See <Topic id="normalization">Normalization</Topic> for details.

Click **Cancel** to exit without creating a target, or **Next step** to go to the next tab.

## Target-Specific Settings

Other tabs contain target-specific configuration options. You can find their details below:

  </TabItem>

<Details summary="File">

The File target writes log messages to destination files and supports field normalization to various logging standards.

<Tabs>
  <TabItem value="file-properties" label="File Properties">
    - **Location** (Optional) - File output directory
    - **File name** (Optional) - File name template. By default, `vmetric.{{.Timestamp}}.{{.Extension}}` 
    - **Type** (Optional) - File format. By default, `json`
    - **Compression** (Optional) - Compression algorithm. By default, `zstd`
    - **Schema** - Data schema for Avro/OCF/Parquet formats
    - **No Buffer** - Toggle to enable/disable buffering. By default, disabled
    - **Format** - Field normalization format (ECS, CIM, ASIM, LEEF, etc.)
  </TabItem>
</Tabs>

For complete configuration details, see <Topic id="targets-console">**this chapter**</Topic>.
</Details>

<Details summary="Microsoft Sentinel">

The **Microsoft Sentinel** target ingests log messages into workspace tables using Data Collection Rules (DCRs) with automatic table selection and field normalization.

<Tabs>
  <TabItem value="azure-properties" label="Azure Properties" default>
    ### Authentication

    - **Managed Identity for Azure** (Conditional) - Enable/disable toggle. If disabled, the user has to specify the following:

    - **Tenant ID** - The unique identifier for your **Azure Active Directory** tenant that contains the Microsoft Sentinel workspace.
    - **Client ID** -  The application (client) identifier for the **Azure AD** app registration used to authenticate with **Microsoft Sentinel** APIs.
    - **Client secret** - The secret key or password for the registered application that provides secure authentication to access Microsoft Sentinel resources.

    Click **Back** to return to the previous step, or **Next step** to go to the final tab.
  </TabItem>

  <TabItem value="stream-properties" label="Stream Properties">
    ### Target Stream Properties

    - **Endpoint** - The Data Collection Rule endpoint URL where log data is sent for processing and routing within **Microsoft Sentinel**
    - **Streams** (Optional) - Dropdown for selecting whether data collection rule streams are automatically detected from incoming data (_Auto_) or manually specified in the configuration (_Manual_)
    - **Buffer Size** (Optional) - Buffer size in bytes. By default, 1 MB
    - **Keep phantom fields** - Toggle for preserving fields that exist in the data but are not explicitly defined in the target schema or mapping configuration. By default, disabled
    - **Drop unknown stream events** - Toggle. By default, disabled
    - **Cache timeout** (Optional) - Stream cache timeout in seconds. By default, 300
    - **Format** (Optional) - Dropdown for field normalization format for the forwarded data. By default, _ASIM_.
  </TabItem>
</Tabs>

Click **Add target** to complete the target creation process.

You are taken back to the list of **Sentinel** targets, notified that the target is created, and you can see the target added to the table.

For complete configuration details, see <Topic id="targets-microsoft-sentinel">**this chapter**</Topic>.
</Details>

<Details summary="Azure Blob Storage">
<Tabs>
  <TabItem value="azure-properties" label="Azure Properties" default>
  ### Authentication

  - **Managed Identity for Azure** - Toggle to enable/disable Azure managed identity. If disabled
    - **Tenant ID** - Azure tenant ID required to be used if managed identity is disabled
    - **Client ID** - Azure client ID required to be used if managed identity is disabled
    - **Client Secret** - Azure client secret required to be used if managed identity is disabled

  ### Azure Properties

  - **Account** - Azure storage account name
  - **Container** (Optional) - Blob container name. By default, `vmetric`

  ### Connection
  
  - **Connection configuration** - Toggle to enable/disable connection settings. When enabled:
    - **Max retry** (Optional) - Maximum number of upload retries. By default, `5`
    - **Retry interval** (Optional) - Base interval between retries in seconds. By default, `10`
    - **Timeout** (Optional) - Connection timeout in seconds. By default, `10`
    - **Max size** (Optional) - Maximum file size in bytes. By default, `0`
  </TabItem>
  <TabItem value="file-properties" label="File Properties">
  ### File Properties

  - **File name** (Optional) - File name template. By default, `vmetric.{{.Timestamp}}.{{.Extension}}`
  - **Type** (Optional) - File format. By default, `json`
  - **Compression** (Optional) - Compression algorithm. Available option: `zstd`
  - **Schema** (Optional) - Data schema for Avro/OCF/Parquet formats
  - **No Buffer** - Toggle to enable/disable unbuffered upload
  - **Format** (Optional) - Field normalization format (e.g., `ECS`, `CIM`, `ASIM`)
  </TabItem>
</Tabs>

For complete configuration details, see <Topic id="targets-azure-blob-storage">**this chapter**</Topic>.
</Details>
