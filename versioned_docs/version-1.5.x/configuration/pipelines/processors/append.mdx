# Append

<span className="theme-doc-version-badge badge badge--primary">Mutate</span><span className="theme-doc-version-badge badge badge--secondary">Elastic Compatible</span>

## Synopsis

Appends one or more values to an array field.

## Schema

```yaml {2,3}
- append:
    description: <text>
    field: <ident>
    value: <any>
    allow_duplicates: <boolean>
    if: <script>
    ignore_failure: <boolean>
    on_failure: <processor[]>
    on_success: <processor[]>
    tag: <string>
```

## Configuration

The following fields are used to define the processor:

|Field|Required|Default|Description|
|---|---|---|---|
|`field`|Y||Field to append the value to|
|`value`|Y||The value to be appended|
|`allow_duplicates`|N|`true`|If `false`, only distinct values are appended|
|`description`|N|-|Explanatory notes|
|`if`|N|-|Condition to run|
|`ignore_failure`|N|`false`|See <Topic id="pipelines-handling-failures">Handling Failures</Topic>|
|`on_failure`|N|-|See <Topic id="pipelines-handling-failures">Handling Failures</Topic>|
|`on_success`|N|-|See <Topic id="pipelines-handling-success">Handling Success</Topic>|
|`tag`|N|-|Identifier|

## Details

If the field exists and is an array, the operation is carried out. If the field exists and is a scalar, the field is converted to an array before the operation. If the field doesn't exist, an array is created containing the provided values.

:::note
The append processor can be combined with <Topic id="processors-foreach">`foreach`</Topic> to append values from another array field.
:::

## Examples

### Basic Usage

<ExampleGrid>
  <CommentCol>
    When appending to a non-existent field...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "name": "document1"
    }
    ```
    ```yaml
    - append:
        field: tags
        value: important
    ```
  </CodeCol>
  <CommentCol>
    a new array is created:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "name": "document1",
      "tags": ["important"]
    }
    ```
  </CodeCol>
</ExampleGrid>

### Multiple Values

<ExampleGrid>
  <CommentCol>
    Multiple values can be appended at once...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "categories": ["books"]
    }
    ```
    ```yaml
    - append:
        field: categories
        value:
          - magazines
          - newspapers
    ```
  </CodeCol>
  <CommentCol>
    which extends the existing array:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "categories": ["books", "magazines", "newspapers"]
    }
    ```
  </CodeCol>
</ExampleGrid>

### Handling Duplicates

<ExampleGrid>
  <CommentCol>
    With `allow_duplicates: false`...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "tags": ["urgent", "important"]
    }
    ```
    ```yaml
    - append:
        field: tags
        value: important
        allow_duplicates: false
    ```
  </CodeCol>
  <CommentCol>
    duplicate values are skipped:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "tags": ["urgent", "important"]
    }
    ```
  </CodeCol>
</ExampleGrid>

### Converting Scalar to Array

<ExampleGrid>
  <CommentCol>
    When appending to a scalar field...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "status": "active"
    }
    ```
    ```yaml
    - append:
        field: status
        value: pending
    ```
  </CodeCol>
  <CommentCol>
    it's automatically converted to an array:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "status": ["active", "pending"]
    }
    ```
  </CodeCol>
</ExampleGrid>

### Using Templates

<ExampleGrid>
  <CommentCol>
    Template values can be used with mustache syntax...
  </CommentCol>
  <CodeCol>
    ```json
    {
      "user_id": "12345",
      "roles": ["user"]
    }
    ```
    ```yaml
    - append:
        field: roles
        value: "role_{{{user_id}}}"
    ```
  </CodeCol>
  <CommentCol>
    which get evaluated during processing:
  </CommentCol>
  <CodeCol>
    ```json
    {
      "user_id": "12345",
      "roles": ["user", "role_12345"]
    }
    ```
  </CodeCol>
</ExampleGrid>
